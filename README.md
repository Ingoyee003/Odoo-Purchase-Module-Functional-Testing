# Odoo Purchase Module – Software Testing Project

This repository contains the complete **Software Quality Assurance (SQA) testing project** conducted on the **Odoo Purchase Module (Version 18.0)**.

The objective of this project was to perform **manual functional testing**, identify system defects, and document the testing workflow following industry-standard QA practices.

---

# Project Overview

The Odoo Purchase Module manages procurement activities such as product purchasing, vendor management, and purchase order processing.  
This testing project evaluates the reliability and functionality of the module through structured testing techniques.

The following major components were tested:

- User Login Validation
- Product Creation
- Vendor Management
- Request for Quotation (RFQ) Management
- Purchase Order Confirmation
- Purchase Order Report Generation

---

# Testing Approach

The testing process followed a structured QA workflow:

1. Requirement Analysis
2. Test Planning
3. Test Scenario Identification
4. Test Case Design
5. Manual Test Execution
6. Defect Identification and Tracking
7. Test Summary Reporting

Testing Type: **Manual Functional Testing**

# Defect Summary

During test execution, several validation and usability defects were identified.

| Bug ID | Description | Severity |
|------|-------------|---------|
BUG-01 | Product creation allows saving without product name | Major |
BUG-02 | Product form does not display total price after price update | Major |
BUG-03 | Tax field not editable during product creation | Minor |
BUG-04 | Duplicate vendor warning not displayed | Major |
BUG-05 | RFQ allows quantity value 0 without validation | Major |
BUG-06 | Purchase order report preview not generated | Major |

All defects were tracked using **Jira Issue Tracking System**.
